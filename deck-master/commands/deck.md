description: Build an editable .pptx deck from a topic, notes, or a rough outline
---

Build a deck about: $ARGUMENTS

Use the `deck-master` skill. Work through it properly rather than jumping
straight to a spec file:

1. Find the argument the deck makes, and write the slide titles first — in
   order, as claims. If the titles do not make an argument on their own, the
   deck is not ready to build.
2. Choose an idiom per slide. Prefer `diagram` wherever the content has stages,
   parts, tiers or a flow.
3. Write the JSON spec, build it, and read every audit finding.
4. Run the six-axis self-critique from `references/audit.md` before handing it
   over. Below 3 on any axis means another pass.

If the topic is thin or ambiguous, ask what the deck is meant to argue and who
is in the room — one round of questions beats a deck that has to be rebuilt.

Report what you built: the slide titles in order, and anything you cut and why.
